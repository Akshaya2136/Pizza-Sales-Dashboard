### 4. Data Source

Source: Raw pizza sales data that I imported and managed in MS SQL Server.

Includes: Order-level details, pizza names, categories, sizes, amounts, timestamps, and sellers.

Workflow:
- Imported raw CSV/XLS data into SQL Server and organized it into relational tables.
- Wrote SQL queries to aggregate, clean, and prepare the metrics for analysis.
- Connected Power BI to SQL Server for interactive preprocessing, modeling, and visualization.

### 5. Features / Highlights

- **Business Problem**  
  Restaurants need to quickly identify bestsellers, minimize wastage, and optimize offerings using real sales data.

- **Goal of the Dashboard**  
  My aim was to deliver an engaging, unique dashboard that:
  - Visualizes critical KPIs (revenue, orders, avg order value, total pizzas sold).
  - Breaks down sales by pizza size and category.
  - Spotlights best and worst sellers with clear visuals.
  - Enables managers to make smarter menu, marketing, and inventory decisions.

- **Walkthrough of Key Visuals**
  - KPIs (Top panel)
    - Total Revenue
    - Average Order Value
    - Total Pizzas Sold
    - Total Orders
    - Average Pizzas Per Order
  - Trend Analysis
    - Daily Total Orders (Bar Chart)
    - Monthly Order Trend (Line Chart)
  - Category & Size Insights
    - Sales % by Pizza Category (Pie/Donut Chart)
    - Sales % by Pizza Size (Pie/Donut Chart)
    - Pizzas Sold by Category (Funnel Chart)
  - Best & Worst Sellers
    - Top 5 pizzas (by revenue, quantity, orders)
    - Bottom 5 pizzas (by revenue, quantity, orders)
  - Filters and Slicers
    - Interactive filtering by category, size, and date for deeper analysis

- **Business Impact & Insights**
  - Menu Optimization: Focus on bestsellers and refine underperforming items.
  - Customer Targeting: Identify preferred pizza sizes or categories.
  - Strategic Decisions: Inform marketing, stocking, and menu update strategies with evidence.
